## 2. 数据流程详解

### 2.1 训练流程 (Training Flow)

```
开始
  │
  ▼
┌─────────────────────────────────────────────────────────────┐
│ 1. 用户上传 FIT 训练记录                                     │
│    输入: .fit 文件列表                                       │
│    输出: 文件路径列表                                        │
└─────────────────────────────────────────────────────────────┘
  │
  ▼
┌─────────────────────────────────────────────────────────────┐
│ 2. 数据验证                                                 │
│    ├─ 检查文件格式 (.fit)                                    │
│    ├─ 检查 GPS 数据 (拒绝跑步机)                             │
│    ├─ 检查海拔数据                                           │
│    └─ 检测重复文件 (基于 time_created)                       │
│    输出: 有效文件列表 + 重复文件警告                          │
└─────────────────────────────────────────────────────────────┘
  │
  ▼
┌─────────────────────────────────────────────────────────────┐
│ 3. 文件筛选                                                 │
│    ├─ 按文件大小排序                                         │
│    └─ 选取最大的 20 个文件                                   │
│    输出: 筛选后的文件列表 (≤20个)                            │
└─────────────────────────────────────────────────────────────┘
  │
  ▼
┌─────────────────────────────────────────────────────────────┐
│ 4. FIT 文件解析                                             │
│    ├─ 读取记录点 (record messages)                          │
│    ├─ 提取时间序列数据                                       │
│    │   ├─ timestamps[]                                       │
│    │   ├─ distances[]                                        │
│    │   ├─ elevations[]                                       │
│    │   └─ heart_rates[] (可选)                               │
│    └─ 输出: 原始时间序列数据                                 │
└─────────────────────────────────────────────────────────────┘
  │
  ▼
┌─────────────────────────────────────────────────────────────┐
│ 5. 海拔滤波处理                                             │
│    ├─ Savitzky-Golay 滤波                                    │
│    │   ├─ window_size = 7                                   │
│    │   ├─ poly_order = 2                                    │
│    │   └─ 输出: smoothed_elevations[]                        │
│    └─ 计算坡度                                               │
│        ├─ grade = (Δele / Δdist) × 100                       │
│        └─ 截断至 [-50%, +50%]                                │
│    输出: 滤波后的海拔和坡度                                   │
└─────────────────────────────────────────────────────────────┘
  │
  ▼
┌─────────────────────────────────────────────────────────────┐
│ 6. 特征提取 (按 200米分段)                                   │
│    对于每个 200米分段:                                        │
│    ├─ 计算分段特征:                                          │
│    │   ├─ speed_kmh = Δdist / Δtime × 3.6                    │
│    │   ├─ grade_pct (当前坡度)                               │
│    │   ├─ rolling_grade_500m (过去500m平均坡度)              │
│    │   ├─ accumulated_distance_km (累计距离)                 │
│    │   ├─ accumulated_ascent_m (累计爬升)                    │
│    │   ├─ absolute_altitude_m (绝对海拔)                     │
│    │   └─ elevation_density (爬升密度)                       │
│    输出: List[SegmentFeatures]                               │
└─────────────────────────────────────────────────────────────┘
  │
  ▼
┌─────────────────────────────────────────────────────────────┐
│ 7. 模型训练                                                 │
│    ├─ 准备训练数据 X, y                                       │
│    │   ├─ X: 特征矩阵 (n_samples × 6_features)              │
│    │   └─ y: 目标变量 (speed_kmh)                            │
│    ├─ LightGBM 训练                                          │
│    │   ├─ num_boost_round = 100                              │
│    │   ├─ learning_rate = 0.05                               │
│    │   └─ 输出: trained_model                                │
│    ├─ 计算特征重要性                                         │
│    └─ 计算能力边界 (P50, P90)                                 │
│    输出: trained_model + feature_importance + P50/P90       │
└─────────────────────────────────────────────────────────────┘
  │
  ▼
训练完成，模型就绪
```

### 2.2 预测流程 (Prediction Flow)

```
开始
  │
  ▼
┌─────────────────────────────────────────────────────────────┐
│ 1. 用户上传 GPX 赛道文件                                     │
│    输入: .gpx 文件                                          │
└─────────────────────────────────────────────────────────────┘
  │
  ▼
┌─────────────────────────────────────────────────────────────┐
│ 2. GPX 文件解析                                             │
│    ├─ 解析 XML 结构                                         │
│    ├─ 提取轨迹点 (trkpt)                                     │
│    │   ├─ lat, lon (经纬度)                                  │
│    │   └─ ele (海拔)                                         │
│    └─ 提取 CP 点 (wpt)                                       │
│        └─ name, lat, lon, ele                                │
│    输出: points[], waypoints[]                               │
└─────────────────────────────────────────────────────────────┘
  │
  ▼
┌─────────────────────────────────────────────────────────────┐
│ 3. GPX 数据处理                                             │
│    ├─ 计算累计距离 (Haversine 公式)                           │
│    ├─ 等距重采样 (20米间隔)                                   │
│    ├─ Savitzky-Golay 滤波                                    │
│    │   ├─ window_size = 7                                   │
│    │   └─ poly_order = 2                                    │
│    └─ 计算坡度并截断 [-45%, +45%]                            │
│    输出: 重采样后的距离、海拔、坡度                            │
└─────────────────────────────────────────────────────────────┘
  │
  ▼
┌─────────────────────────────────────────────────────────────┐
│ 4. 创建预测分段 (200米/段)                                   │
│    对于每个 200米分段:                                        │
│    ├─ 计算特征 (与训练相同)                                  │
│    ├─ 匹配最近的 CP 点 (500米内)                             │
│    └─ 计算本段爬升/下降                                      │
│    输出: List[SegmentFeatures] with CP names                │
└─────────────────────────────────────────────────────────────┘
  │
  ▼
┌─────────────────────────────────────────────────────────────┐
│ 5. 模型预测                                                 │
│    对于每个分段:                                              │
│    ├─ LightGBM 预测基础速度                                   │
│    │   predicted_speed = model.predict(features)            │
│    ├─ 应用努力程度系数                                        │
│    │   predicted_speed *= effort_factor                      │
│    ├─ 应用物理约束:                                          │
│    │   ├─ VAM 限制 (坡度 > 15%)                              │
│    │   ├─ 距离外推惩罚                                        │
│    │   └─ 爬升外推惩罚                                        │
│    └─ 限制速度范围 [1.0, P90×1.1]                             │
│    输出: 每段的预测速度                                       │
└─────────────────────────────────────────────────────────────┘
  │
  ▼
┌─────────────────────────────────────────────────────────────┐
│ 6. 计算预测结果                                             │
│    ├─ 计算每段时间: time = distance / speed                  │
│    ├─ 计算累计时间: cumulative_time += time                 │
│    ├─ 计算总时间: total_time = Σ(time)                       │
│    ├─ 计算配速: pace = total_time / total_distance           │
│    └─ 计算平均速度: avg_speed = total_distance / total_time  │
│    输出: PredictionResult                                    │
└─────────────────────────────────────────────────────────────┘
  │
  ▼
┌─────────────────────────────────────────────────────────────┐
│ 7. 生成显示数据                                             │
│    ├─ 构建分段配速表                                          │
│    │   ├─ 起点 (0km)                                         │
│    │   ├─ CP 点或 5km 间隔                                    │
│    │   └─ 终点                                               │
│    ├─ 生成战术建议                                            │
│    └─ 生成难度预警                                            │
│    输出: 可显示的结果数据                                     │
└─────────────────────────────────────────────────────────────┘
  │
  ▼
┌─────────────────────────────────────────────────────────────┐
│ 8. 生成报告                                                 │
│    ├─ HTML 报告 (精美样式)                                   │
│    └─ TXT 报告 (纯文本)                                      │
│    输出: 报告文件                                            │
└─────────────────────────────────────────────────────────────┘
  │
  ▼
预测完成
```

## 4. 关键数据结构

### 4.1 SegmentFeatures (分段特征)

**用途**: 训练和预测的基本数据单元

```python
@dataclass
class SegmentFeatures:
    # 目标变量
    speed_kmh: float                  # 平均速度

    # 地形特征
    grade_pct: float                  # 当前坡度
    rolling_grade_500m: float         # 过去500米平均坡度

    # 疲劳/消耗特征
    accumulated_distance_km: float    # 累计距离
    accumulated_ascent_m: float       # 累计爬升

    # 生理/环境特征
    absolute_altitude_m: float        # 绝对海拔

    # 辅助特征
    elevation_density: float          # 爬升密度
    is_climbing: bool                 # 是否爬升
    is_descending: bool               # 是否下降

    # 显示用字段
    segment_ascent_m: float = 0       # 本段爬升
    segment_descent_m: float = 0      # 本段下降
    cp_name: str = ""                 # CP点名称
```

**数据流**: FIT解析 → 滤波处理 → 特征提取 → 训练数据

## 5. 关键算法说明

### 5.1 Savitzky-Golay 滤波

**目的**: 去除海拔噪声，保留真实地形特征

**公式**:
```
平滑后的海拔[i] = Σ(权重[k] × 原始海拔[i+k])
```

**参数**:
- 窗口大小: 7 点
- 多项式阶数: 2

**效果**: 去除 GPS 噪声，保留山体起伏曲线

---

### 5.2 Haversine 距离计算

**目的**: 计算地球表面两点间的大圆距离

**公式**:
```
a = sin²(Δlat/2) + cos(lat1) × cos(lat2) × sin²(Δlon/2)
c = 2 × asin(√a)
d = R × c  (R = 6371000 米)
```

**用途**: GPX 轨迹点距离计算

---

### 5.3 VAM (垂直上升速度)

**目的**: 限制陡坡预测速度，防止不切实际

**公式**:
```
VAM = 水平速度 × 1000 × 坡度 / 100
    = 水平速度 × 10 × 坡度
```

**限制**: VAM ≤ 1000 m/h

**触发**: 坡度 > 15%

---

### 5.4 能力边界计算

**目的**: 量化用户的速度分布

**P50 (中位数)**:
```
筛选平路段数据 (-5% ~ 5%)
P50 = percentile(速度, 50)
```

**P90 (第90百分位)**:
```
筛选平路段数据 (-5% ~ 5%)
P90 = percentile(速度, 90)
```

**努力范围**:
```
effort_range = P90 / P50
```

## 7. 性能优化要点

### 7.1 数据处理优化

- ✅ 使用 numpy 向量化计算
- ✅ 限制处理文件数量 (最多20个)
- ✅ 使用高效的插值算法 (scipy interp1d)
- ✅ 避免不必要的数据复制

### 7.2 模型训练优化

- ✅ LightGBM 自动优化
- ✅ 限制迭代次数 (100轮)
- ✅ 使用适当的数据类型 (float32)

### 7.3 界面渲染优化

- ✅ 使用 st.cache_data 缓存计算结果
- ✅ 分页显示大量数据
- ✅ 延迟加载非关键内容
