## 1. 数据输入模块

### 1.1 GPX 赛道文件上传

**功能描述**: 用户上传比赛的 GPX 路线文件

**输入规格**:
- 文件格式: `.gpx`
- 必须包含字段:
  - 轨迹点 (`trkpt`): 经度、纬度
  - 海拔 (`ele`): 海拔数据（米）
- 可选字段:
  - 航点 (`wpt`): CP 点信息，包含名称、经纬度、海拔

**处理流程**:
1. 解析 XML 结构
2. 提取所有轨迹点
3. 提取 CP 点（如果有）
4. 计算累计距离（使用 Haversine 公式）

**关键代码位置**: `app.py:115-120`, `core/predictor.py:722-775`

---

### 1.2 FIT 训练记录上传

**功能描述**: 用户上传历史训练数据（支持多文件）

**输入规格**:
- 文件格式: `.fit`
- 支持压缩格式: `.zip`, `.gz`
- 必须包含字段:
  - `timestamp`: 时间戳
  - `distance` 或 `enhanced_distance`: 累计距离（米）
  - `altitude` 或 `enhanced_altitude`: 海拔（米）
- 可选字段:
  - `heart_rate`: 心率
  - `position_lat/lon`: GPS 坐标（用于判断室内/室外）

**处理流程**:
1. 检测文件格式（原始/ZIP/GZIP）
2. 解压缩（如果需要）
3. 解析 FIT 记录点
4. 提取时间序列数据

**关键代码位置**: `app.py:123-129`, `core/predictor.py:265-393`

**文件数量要求**:
- 最少: 5 个文件
- 推荐: 15-20 个精品文件
- 系统自动选取: 最多使用 20 个最大的文件

---

### 1.3 努力程度调节

**功能描述**: 用户通过滑块调节比赛时的努力程度

**输入规格**:
- 类型: 滑块输入
- 范围: 0.8 - 1.2
- 默认值: 1.0
- 步长: 0.01

**数值含义**:
| 值范围 | 含义 | 对应能力 |
|--------|------|----------|
| 0.8-0.9 | 保守策略 | 低于 P50 |
| 1.0 | 平均水平 | P50（中位数速度）|
| 1.1-1.2 | 比赛状态 | 接近 P90（第90百分位）|

**应用方式**:
```
预测速度 = 模型基础预测 × effort_factor
```

**关键代码位置**: `app.py:142-158`, `core/predictor.py:191-254`

## 2. 数据验证模块

### 2.1 GPX 文件验证

**功能描述**: 验证 GPX 文件的完整性和有效性

**验证项**:
| 检查项 | 说明 | 失败处理 |
|--------|------|----------|
| 文件格式 | XML 解析成功 | 拒绝文件 |
| 轨迹点存在 | 至少 1 个 trkpt | 拒绝文件 |
| 海拔数据 | ele 字段存在 | 警告，继续运行 |
| 坐标范围 | 纬度 [-90,90], 经度 [-180,180] | 拒绝文件 |
| 点数量 | 建议至少 50 个点 | 警告 |

**关键代码位置**: `data/data_validator.py:35-127`

---

### 2.2 FIT 文件验证

**功能描述**: 验证 FIT 文件的有效性

**验证项**:
| 检查项 | 说明 | 失败处理 |
|--------|------|----------|
| 文件头 | 包含 `.FIT` 魔数 | 警告，继续尝试 |
| GPS 数据 | position_lat 存在 | 拒绝（可能是跑步机）|
| 海拔数据 | altitude 字段存在 | 警告 |
| 记录点数 | 建议至少 60 个点 | 警告 |

**关键代码位置**: `data/data_validator.py:130-204`

---

### 2.3 重复数据检测

**功能描述**: 自动检测并过滤重复上传的训练文件

**检测规则**:
1. **FIT 文件**: 基于 `time_created` 字段（活动创建时间）
2. **JSON 文件**: 基于第一条记录的时间戳
3. **文件名相同**: 也视为重复

**处理方式**:
- 保留第一次出现的文件
- 后续重复文件自动过滤
- 显示警告信息，列出所有重复文件

**关键代码位置**: `app.py:250-325`

## 3. 数据处理模块

### 3.1 海拔滤波处理

**功能描述**: 使用 Savitzky-Golay 滤波器去除海拔噪声

**滤波配置**:

#### GPX 赛道配置
```python
GPX_CONFIG = {
    'resample_spacing_m': 20,      # 20米重采样
    'window_size': 7,              # 窗口大小（7点 ≈ 140米）
    'poly_order': 2,               # 多项式阶数
    'max_grade_pct': 45.0,         # 坡度截断 ±45%
    'resample_required': True      # 必须重采样
}
```

#### FIT 训练记录配置
```python
FIT_CONFIG = {
    'resample_spacing_m': None,    # 不重采样
    'window_size': 7,              # 窗口大小（7-10秒）
    'poly_order': 2,               # 多项式阶数
    'max_grade_pct': 50.0,         # 坡度截断 ±50%
    'resample_required': False     # 不需要重采样
}
```

**处理流程**:
1. 重采样（GPX）: 等距重采样为 20 米间隔
2. 滤波: Savitzky-Golay 滤波
3. 坡度计算: 基于滤波后海拔计算坡度
4. 坡度截断: 限制在 ±45%（GPX）或 ±50%（FIT）

**关键代码位置**: `core/utils.py:36-123`, `core/gpx_filter.py:113-186`

## 4. 预测模型模块

### 4.1 LightGBM 模型配置

**功能描述**: 使用 LightGBM 梯度提升框架进行速度预测

**模型参数**:
```python
params = {
    'objective': 'regression',     # 回归任务
    'metric': 'mae',               # 平均绝对误差
    'num_leaves': 31,              # 叶子节点数
    'learning_rate': 0.05,         # 学习率
    'feature_fraction': 0.9,       # 特征采样比例
    'bagging_fraction': 0.8,       # 数据采样比例
    'bagging_freq': 5,             # 采样频率
    'min_data': 1,                 # 最小叶子数据
    'num_boost_round': 100         # 迭代次数
}
```

**训练特征** (6个):
1. `grade_pct` - 当前坡度
2. `rolling_grade_500m` - 过去500米平均坡度
3. `accumulated_distance_km` - 累计距离
4. `accumulated_ascent_m` - 累计爬升
5. `absolute_altitude_m` - 绝对海拔
6. `elevation_density` - 爬升密度

**关键代码位置**: `core/predictor.py:75-168`

---

### 4.2 能力边界量化 (P50/P90)

**功能描述**: 基于训练数据统计用户的能力边界

**计算方法**:
- **P50 速度**: 平路段（坡度 -5% ~ 5%）速度的中位数
- **P90 速度**: 平路段速度的第 90 百分位
- **努力范围**: P90 / P50 比值

**解读**:
| P90/P50 比值 | 解读 |
|-------------|------|
| 1.10-1.15 | 训练强度稳定，提升空间有限 |
| 1.15-1.25 | 正常范围，比赛可提升 15-25% |
| >1.25 | 训练强度差异大，可能存在"划水"训练 |

**关键代码位置**: `core/predictor.py:154-163`

---

### 4.3 物理约束机制

**功能描述**: 防止模型产生不切实际的预测

#### VAM (垂直上升速度) 限制
```
VAM = 水平速度 × 1000 × 坡度 / 100
    = 水平速度 × 10 × 坡度
```

**限制**: VAM ≤ 1000 m/h（大神级）

**触发条件**: 坡度 > 15% 的陡坡路段

**惩罚系数**: `vam_penalty = vam / 1000`，预测速度除以惩罚系数

#### 外推惩罚机制
当预测超出训练数据范围时应用惩罚:

**距离外推**:
```python
if 累计距离 > 训练最大距离:
    超出比例 = 累计距离 / 训练最大距离
    惩罚系数 = 1 + (超出比例 - 1) × 0.3  # 每超出10%慢3%
    预测速度 /= 惩罚系数
```

**爬升外推**:
```python
if 累计爬升 > 训练最大爬升:
    超出比例 = 累计爬升 / 训练最大爬升
    惩罚系数 = 1 + (超出比例 - 1) × 0.2  # 每超出10%慢2%
    预测速度 /= 惩罚系数
```

**关键代码位置**: `core/predictor.py:218-242`

### 4.4 回退模型

**功能描述**: 当 LightGBM 不可用或数据不足时使用的简化模型

**触发条件**:
1. LightGBM 未安装
2. 训练数据少于 10 个分段

**预测逻辑**:
- 按坡度分组统计平均速度
  - 爬坡（坡度 > 5%）
  - 平路（-5% ≤ 坡度 ≤ 5%）
  - 下坡（坡度 < -5%）
- 根据当前路段坡度选择对应组别速度

**关键代码位置**: `core/predictor.py:170-189, 244-254`

### 5.2 分段配速表

**功能描述**: 显示详细的分段配速信息

**显示规则**:
1. **第一行**: 永远是起点（距离 0km）
2. **中间行**:
   - 如果 GPX 有 CP 点: 按所有 CP 点显示
   - 如果 GPX 无 CP 点: 按 5km 间隔显示
3. **最后一行**: 永远是终点

**表格列**:
| 列名 | 说明 |
|------|------|
| 位置 | 起点/CP点/终点 |
| 本段距离(km) | 从上一位置到当前位置的距离 |
| 距离(km) | 当前位置的累计距离 |
| 本段爬升(m) | 本段的累计爬升 |
| 本段下降(m) | 本段的累计下降 |
| 本段时间 | H:MM:SS |
| 累计时间 | H:MM:SS |
| 平均速度 | X.X km/h |

**CP 点匹配规则**:
- 距离 CP 点 500 米以内的分段标记该 CP 名称
- 去重：每个 CP 名称只显示第一次出现的位置

**关键代码位置**: `app.py:582-727`, `app_splits.py:5-157`

### 5.3 战术建议生成

**功能描述**: 基于赛道和用户特征生成战术建议

**配速策略**:
- 基于 `feature_importance` 分析主要影响因素
  - 如果 `grade_pct` > `accumulated_distance_km`: 坡度影响大，建议陡坡保守
  - 否则: 疲劳影响大，建议前半程控制速度
- 基于 `elevation_density` 判断赛道难度
  - > 80 m/km: 极高难度，前 10km 建议配速比预测慢 10%
  - > 50 m/km: 有一定难度，建议均匀分配体力

**补给建议**:
- > 6 小时: 携带能量胶 N 支，每 45-60 分钟补充
- 3-6 小时: 携带能量胶 N 支，每 45 分钟补充
- < 3 小时: 短距离，少量补给

**关键代码位置**: `app.py:729-760`, `core/report_generator.py:655-735`

## 8. 数据类型定义

### 8.1 核心数据结构

**SegmentPrediction** (单段预测):
```python
@dataclass
class SegmentPrediction:
    segment_id: int              # 分段编号
    start_km: float              # 起始距离
    end_km: float                # 结束距离
    distance_km: float           # 分段距离
    grade_pct: float             # 坡度
    altitude_m: float            # 海拔
    predicted_speed_kmh: float   # 预测速度
    predicted_time_min: float    # 预测时间
    cumulative_time_min: float   # 累计时间
    difficulty_level: str        # 难度等级
    grade_type: str              # 地形类型
    ascent_m: float              # 本段爬升
    descent_m: float             # 本段下降
    cp_name: str                 # CP点名称
```

**PredictionResult** (完整预测):
```python
@dataclass
class PredictionResult:
    total_time_min: float        # 总时间（分钟）
    total_time_hm: str           # 总时间（H:MM:SS）
    pace_min_km: float           # 配速
    speed_kmh: float             # 速度
    total_distance_km: float     # 总距离
    total_ascent_m: float        # 总爬升
    total_descent_m: float       # 总下降
    elevation_density: float     # 爬升密度
    segments: List[SegmentPrediction]  # 分段列表
    feature_importance: Dict     # 特征重要性
    model_confidence: float      # 模型置信度
    effort_level: str            # 努力程度
    training_stats: Dict         # 训练统计
    warnings: List[str]          # 警告信息
```

**关键代码位置**: `core/types.py:52-143`
