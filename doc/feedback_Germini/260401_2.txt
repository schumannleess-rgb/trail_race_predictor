```python
def predict_with_effort(self, segment_data, effort_factor):
    # 1. 模型预测平均表现
    base_speed = self.model.predict(segment_data)
    
    # 2. 应用努力程度系数 (例如 0.8 ~ 1.2)
    adjusted_speed = base_speed * effort_factor
    
    # 3. 物理锁死：即便努力程度再高，也不能超过 VAM 极限
    # 且不能超过历史 P95 的绝对速度（防止模型跑飞）
    final_speed = min(adjusted_speed, self.vam_limit, self.p95_absolute_speed)
    return final_speed
```

### 2. 努力程度 (Effort) 的科学量化与代码实现
我们要去除主观的“档位”，改为基于数据分布的连续调节。

#### A. 能力边界获取 (Profiling)
在训练模型的同时，程序会自动计算两个核心指标作为滑块的物理锚点：
* **P50 (中位数速度)：** 代表用户历史记录中的平均努力程度（即滑块的 1.0 处）。
* **P90 (前 10% 速度)：** 代表用户在竞技状态下的潜力（通常建议作为滑块的 1.1 - 1.15 处）。
* **心率关联：** 若 FIT 包含心率，计算平均心率与最大心率的比例，辅助验证 P90 采样点是否为真实的“高强度”片段。

#### B. 预测模型变更 (Inference)
在 `predictor.py` 中，速度预测逻辑将演变为：
$$V_{final} = \text{Model}(Grade, Dist, \dots) \times \text{Effort\_Factor}$$
同时保留 **VAM 物理锁**（如 1000 m/h 限制），确保即便努力程度拉满，预测速度也不会超过人类生理极限。
