### 1.3 努力程度滑块

- [ ] **参数确认**
  - [ ] 最小值: 0.8
  - [ ] 最大值: 1.2
  - [ ] 默认值: 1.0
  - [ ] 步长: 0.01

- [ ] **数值含义**
  - [ ] 0.8-0.9 = 保守策略
  - [ ] 1.0 = 平均水平 (P50)
  - [ ] 1.1-1.2 = 比赛状态 (接近 P90)

- [ ] **显示反馈**
  - [ ] < 0.95: 显示保守策略提示
  - [ ] 0.95-1.05: 显示平均水平提示
  - [ ] > 1.05: 显示比赛状态提示

### 3.2 能力边界计算

- [ ] **P50 计算** (不可修改)
  - [ ] 筛选条件: 平路段 (坡度 -5% ~ 5%)
  - [ ] 统计方法: 中位数 (percentile 50)

- [ ] **P90 计算** (不可修改)
  - [ ] 筛选条件: 平路段 (坡度 -5% ~ 5%)
  - [ ] 统计方法: 第 90 百分位 (percentile 90)

- [ ] **备用方法** (当无平路数据时)
  - [ ] 使用全部数据计算 P50 和 P90

**确认签名**: _________________ 日期: _______

---

### 3.3 物理约束机制

- [ ] **VAM 限制** (不可修改)
  - [ ] 触发条件: 坡度 > 15%
  - [ ] 限制值: VAM ≤ 1000 m/h
  - [ ] 惩罚公式: `predicted_speed /= (vam / 1000)`

- [ ] **距离外推惩罚** (不可修改)
  - [ ] 触发条件: 累计距离 > 训练最大距离
  - [ ] 惩罚系数: `1 + (超出比例 - 1) × 0.3`
  - [ ] 应用方式: `预测速度 /= 惩罚系数`

- [ ] **爬升外推惩罚** (不可修改)
  - [ ] 触发条件: 累计爬升 > 训练最大爬升
  - [ ] 惩罚系数: `1 + (超出比例 - 1) × 0.2`
  - [ ] 应用方式: `预测速度 /= 惩罚系数`

## 模块 4: 结果显示

### 4.1 分段配速表规则

- [ ] **第一行** (不可修改)
  - [ ] 永远显示"起点"
  - [ ] 距离: 0 km
  - [ ] 本段数据: 全部显示 "-"

- [ ] **中间行** (不可修改)
  - [ ] 有 CP 点: 显示所有 CP 点
  - [ ] 无 CP 点: 每 5 km 显示一行
  - [ ] CP 点去重: 每个名称只显示第一次

- [ ] **最后一行** (不可修改)
  - [ ] 永远显示"终点"
  - [ ] 即使终点距离与上一行相同也要显示

- [ ] **CP 点匹配规则** (不可修改)
  - [ ] 匹配距离阈值: 500 米
  - [ ] 每个分段只匹配最近的 CP
